## 📌 1. What — 我们要做什么？

本项目的目标是创建一个名为 **`file_manager.py`** 的 Python 脚本，能够自动执行以下操作：

1. **创建一个新的文件夹 `data/`**
   如果不存在，则自动生成。

2. **在 `data` 文件夹中创建 5 个文本文件**
   文件名依次为：

   * `file1.txt`
   * `file2.txt`
   * `file3.txt`
   * `file4.txt`
   * `file5.txt`

3. **为每个文件写入一个三列的表格数据**
   表格包含字段：

   * 编号
   * 名称
   * 描述
     内容由脚本自动生成虚拟数据。

4. **读取所有文件内容，并带标题输出到控制台**
   每个文件在打印内容前需要输出文件名作为标题。

5. **从每个文件中提取“名称”列**
   将所有文件中的名称汇总成一个新的表格。

6. **将汇总表保存为 `summary.txt`**
   位于项目根目录或指定位置（可根据实现调整）。

## 🎯 2. Why — 为什么要做这些？

这些任务旨在让学生掌握 Python 中文件系统相关的核心技能，包括：

### ● **学习文件与文件夹操作**

* 使用 `os`、`pathlib` 创建目录
* 检查文件是否存在
* 自动批量生成文件

### ● **理解文本文件读写**

* 使用 `open()` 写入结构化内容
* 处理文本格式数据（如 CSV 风格）
* 使用循环处理多文件读写流程

### ● **练习结构化数据的组织方式**

* 表格写入（编号、名称、描述）
* 提取特定列（名称）
* 汇总与整合数据

### ● **培养自动化批处理思维**

此作业模拟了自动化脚本如何批量：

* 生成资源
* 解析内容
* 整理结果

这是进入更复杂的数据处理、日志提取、批处理脚本的基础。

### ● **引导学生编写可复用的小工具**

最终的 `file_manager.py` 是一个小型、完整的实用工具：

* 有输入
* 有输出
* 有逻辑流程
* 可扩展
  这符合从“写脚本 → 写组件 → 写项目”的渐进式学习路径。
